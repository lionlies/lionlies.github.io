Rails 101 note 2

Posted on 2016-09-22   |  

  • 保持一個原則： 動到除了 app 資料夾以外的檔案，都要重跑 $ rails s

  • 在 MVC 架構下要做什麼東西出來，是瀏覽器端跟網站的 server 之前存取資料的動作，每當我們輸入網址，點擊某個連結，都是代表

  • 我們從瀏覽器端發出請求(request)指令到網站 server 端，後者再以 request 的內容來回應資料給前者，也就是網頁資料的呈現。

  • 絕大多數 HTTP request 都是 GET，也就是跟 Server 端請求讀取某個網址的資料。
    有要更動到 server 裡資料庫的資料，就會用到另外的三個動作。
    當我們要用 Rails 做一個有 CRUD 功能(Create, Read, Update, Destroy)的討論版，正好就對應到 HTTP Verb 上的四個動詞。

  • 『網頁』跟『網站』製作最大的差別，在於前者只有 純讀取 資訊的功能
    後者可以運作 CRUD 功能，讓系統功能強大
    但從前者走向後者，整個系統的規劃與運作就會變得非常龐大與複雜
    為了方便可以區分功能與團隊分工合作將一個網站專案建立起來
    近代的網站開發走向了 MVC 架構：
    優點是什麼？
    可以把整個專案(Project)拆分
    負責做前端的人只做 Views
    寫流程控制的專心寫 Controllers
    所有資料相關集中在 Models
    三者之間不互相打架，出問題與狀況也容易追出問題的位置
    Ruby on Rails 就是用 Ruby 這個程式語言寫出來的網站 MVC 框架(Framework)

  • 我們可以任意設定 Action 的命名，只要 Request 正確，就有相對應的動作
    但太過自由的設計，會導致專案開發的災難

    • 若無對 Request 動作分類做統一的規範，一樣的 index action，可能會有不一樣的內容
      導致團隊夥伴在開發時必須花時間閱讀程式碼的內容，也許這個 action 僅需花 1 分鐘
      可是 10 個夥伴，就會花費整個團隊 10 分鐘的時間在這件事上面
    • 若夥伴或你本人剛做好一個新的 action ，卻發現這個功能之前就做過一個類似的…
      未來若又要沿用這個 action 做延伸功能，要嘛再重讀一次之前的程式碼，不然又要再重做一個新的 action …
    • 資訊安全問題，在新增，更新，刪除某筆資料時，必須要有驗證使用者權限的機制
      延續 2nd ，要怎麼知道之前的夥伴是否有做？(花時間重讀) 還是乾脆做一個新的(重複動作)
      又要怎麼讓資訊安全機制不會變成龐大工程，讓團隊精力專心在專案開發上？

  • 結論：
    最佳解決方案： RESTful

延伸閱讀：CRUD

CRUD in Rails

  • CRUD(Create/Read/Update/Destroy)是網站最基礎的結構，Rails中建立CRUD結構的規範是 MVC 和 RESTful。MVC是Model, View, Controller, 這樣的架構可以把前後端和資料庫的工作切開，各做各的，比較好開發和管理。RESTful則是透過一種HTTP verb搭配一種helper對應到特定的action,

    1. GET + groups_path => index action
    2. GET + group_path(group) => show
    3. GET + edit_group_path(group) => edit
    4. GET + new_group_path => new
    5. POST + groups_path => create
    6. PUT + group_path(group) => update
    7. DELETE + group_path(group) => destroy

其他的組合則不會有任何動作，這樣不管瀏覽器端傳來什麼樣的request, routing都會直接幫我們導向對應的action去運作後續流程，並且因為只有create, update destroy這三個action才會動到server端的資料，也就一併解決了資安問題。

目前 helpers directory 內有 posts_helper.rb 以及 account/posts_helper.rb 兩個 helper.rb，經過測試，

  1. 在 posts_helper.rb 以及 account/posts_helper.rb 中寫入名字相同內容不同的兩個 helpers, posts_helper.rb 的優先。
