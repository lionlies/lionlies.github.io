Posted on 2016-09-22   |  

Misc.

Controller, model各會放入什麼邏輯的methods, 這部分常常會搞混。

初學沒經驗的時候，先把所有的methods都寫在controller沒關係，要多噁心就多噁心，之後整理code的時候再分解到model去。

有沒有什麼比較固定的開發流程？整套教材到目前為止，有時候先做controller, routing, model, view, 有時候先model, routing, controller, view, 有時候又再不一樣，都搞混了。

xdite 寫 code 流程：routing -> controller -> view -> model -> partial -> helper
每個人的做法不同，xdite 習慣先讓 view 可以讀，那先設好 routing 之後，controller 和 view 就能動了，這時再去寫 controller 看要操作什麼東西，以及寫 view 看要寫什麼東西。如果 controller 變得肥了一點，xdite 就把裡面的東西 refactor 到 model；如果 view 變肥了，就把它切成 partial；如果 partial 裡面有些醜醜的 ruby logic code, 就再把它 refactor 到 helper.

warnings是怎麼顯示的？

flash[:warning] = “message” (controller)
notice: “message” (controller)
alert_types = { notice: :success, alert: :danger } (application helper)

為什麼database不能放在github上？

  1. DB 檔案通常很大
  2. DB 需要帳密，有安全性問題
  3. DB 的內容可能有私人資料
